Current project documentation

ColorMill

ColorMill is a browser-based C/raylib/WASM simulator for exploring how pigment disperses through silicone on a two-roll mill. The project is currently a CPU grid debugger: it is intentionally simple, inspectable, and domain-driven before becoming a polished visual or GPU shader simulation.

Current design rule: transport happens through the visible material band, but active mixing primarily happens at the nip horizon where the rollers press the material.

Current layout

Front-on view

The simulation looks at the mill from the front, where the two rollers meet. The grid represents the visible working band rather than the entire machine.

Feed / entry

The top region is where material approaches the nip. It is mostly transport, not the main place where color is blended.

Nip / mixing horizon

A narrow band around the roller gap is the primary active mixing zone. This is where pressure, shear, compression, and dispersion are strongest.

Return / transport

The lower visible band carries material after it has passed the nip. It should preserve color state more than continuously blur it.

What exists in the code today

AreaCurrent implementation
RuntimeC + raylib compiled to WebAssembly with Emscripten. GitHub Pages deploys the built dist/ folder.
Simulation gridA 128 x 72 CPU grid. Each cell tracks material amount, red pigment, blue pigment, velocity X/Y, a band mask, and a mix mask.
Band maskDefines where silicone is allowed to exist. The current shape approximates top feed, nip, and a lower/front curtain or return band.
Mix maskDefines the active nip horizon. Diffusion/mixing is multiplied by this mask so color changes mostly at the nip.
Velocity fieldGenerated from a simplified two-roller front view: roller surfaces, downward nip pull, lateral shear near the gap, and slower return transport lower in the band.
RenderingGrid cells render to a raylib texture. The UI includes debug modes for mixed color, material, pigments, velocity, band mask, and mix mask.

Domain vocabulary

TwoRollMill

The machine: two rollers, adjustable gap, speed settings, surface grip, and process settings.

Roller

A rotating cylinder with diameter, width, RPM, surface speed, direction, grip/friction, and optionally temperature.

Nip

The closest space between rollers. This is the main transformation event: compression, shear, stretching, and pigment dispersion.

MaterialBatch

The silicone load: weight, base color, opacity/translucency, viscosity, elasticity, tackiness, pigment loads, and current dispersion state.

SimulationGrid

The front-view numerical model that stores material and pigment state per cell.

Pass

One journey of material through the nip. Repeated passes, often with cutting/folding, gradually homogenize color.

Current limitations

  • The current grid is still a 2D approximation, not true 3D roller mechanics.
  • Material does not yet conserve total mass perfectly.
  • There is no explicit layer stack, so fold/lamination is not yet represented accurately.
  • There is no speed ratio yet; both roller effects are simplified into one surface speed.
  • Color rendering is simple red/blue pigment over a generic base, not yet a real optical/translucency model.

Next short-term work

  1. Fix deployment reliability. Keep the GitHub Pages build deterministic and make simulator/docs easy to open.
  2. Stabilize the grid model. Tune band mask, mix mask, and velocity field until it matches the front-on reference view.
  3. Add machine settings. Introduce explicit roller diameter, gap, RPM, width, surface speed, and speed ratio objects.
  4. Add material settings. Start with batch weight, base color, opacity/translucency, viscosity, tackiness, and pigment loads.
  5. Add pass/fold operations. Model cut, fold, and repeated passes as domain events that affect layer distribution before entering the nip.

Long-term vision

The long-term goal is a domain-driven mill mixing simulator where the user can configure a real-ish two-roll mill, choose silicone material properties and pigment loads, then observe how repeated passes through the nip transform a streaky batch into a more homogeneous material.

  1. Phase 1: CPU debugger. A clear grid view with masks, velocity, pigment concentration, and nip-only mixing.
  2. Phase 2: Domain model. Machine, roller, nip, material batch, pigment load, pass, cut, fold, and banding objects.
  3. Phase 3: Better material physics. Improve mass conservation, thickness, shear/stretch, layer merging, and dispersion state.
  4. Phase 4: Better appearance. Render opaque/translucent silicone, base colors, pigment concentration, and streak depth more believably.
  5. Phase 5: GPU path. Port the stable CPU grid algorithm to shader/texture ping-pong for larger grids and smoother interaction.
  6. Phase 6: Review/workbench UI. A full page where diagrams, settings, runs, notes, and references are all accessible for discussion.
